# ADR-0042 implementation — Slices 1–2 (2026-07-19)

Measure-first staged migration to qualified-key index registration (the
approach ratified after the gate: qualify declaration keys + a short→qualified
alias table with ambiguity-collapse; keep internal REFERENCES short and only
add lexical resolution where measurement demands it — ADR step 3).

## Slice 1 — qualified substrate (purely additive)

`rigor-index/src/rbs.rs` only. Registers every class/module ALSO under its
fully-qualified name (lexical enclosing prefix + the node's namespace path +
leaf) alongside the UNCHANGED short-key `classes` map, plus a
`short_to_qualified` alias table. Four read accessors (`knows_qualified_class`,
`resolve_short_unambiguous` = the ambiguity-collapse-to-nothing rule,
`qualified_declares_singleton`/`_instance`). No consumer calls them yet.

Invariant proven: single-file additive diff (removed lines are pure
signature/call-site threading); whole workspace green (+5 tests); harness
byte-unchanged (208 matched / 0 FP; snapshots 0 git changes). The Util
MERGE-collision is SPLIT at the substrate (ERB::Util / CGI::Util distinct,
unit-tested method-disjoint) while the short key "Util" still maps to the
merged composite — nothing observable moved.

## Slice 2 — qualified singleton (class-method) witnessing

Wires the substrate into class-method typo detection for a namespaced
receiver:
- `class_has_singleton_method` transparently routes a qualified name absent
  from the short-key map to `qualified_class_has_singleton_method` (own
  singleton surface + extends' instance methods + the shared base-object
  surface, same complete-surface silence gate). Top-level names untouched.
- the `ConstantRead` typer arm mints a `Singleton` for an unambiguous
  namespaced constant, gated on the qualified registry (NOT the
  defect-2-refusing `knows_toplevel_class`); the project-shadow gate still
  wins. SourceIndex Pass-2 registers the qualified name for round-trip.

Measured: fixture 68's six SINGLETON cases match the oracle byte-for-byte,
including the MERGE-collision split (no cross-contamination). Live 208→213
matched, 9→4 gaps, **0 FP**; snapshot re-baselined; fp_audit **0 FP** on
gitlab lib (4676) + app/models, mastodon models + lib, conference-app;
**gitlab UM gaps 148→145** (the namespaced-singleton cluster).

**The survey sweep earned its keep** (measure-first, not a formality): the five
core corpora were 0 FP but the survey found two over-firing modes the core set
does not exercise —
1. qualified CLASSES (`singleton(Gem::Specification)`, 36 FPs on dependabot):
   a class inherits class methods down a superclass chain this slice does not
   walk (references stay short pending ADR step 3) → restrict qualified
   singleton witnessing to MODULES (self-contained surface);
2. singleton ALIASES (`ERB::Util.h` = `alias self.h self.html_escape`, rails)
   → resolve the alias to its target.
Re-measured against a master worktree binary: dependabot 38→2 (==master),
rails 72→67 (==master), mastodon 30==30 (pre-existing noise). +3 index test
assertions.
